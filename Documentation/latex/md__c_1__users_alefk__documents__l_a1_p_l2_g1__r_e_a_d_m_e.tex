\item {\itshape Criação dos módulos}\+: A princípio, foram criados 4 módulos\+: 1 para a camada de dados, 2 para a lógica do programa e 1 para a camada de interface. Decidiu-\/se dividir a lógica do programa em dois módulos para se ter uma melhor ideia de quais funções modificam o estado.
\item {\itshape Inicializar o estado do jogo}\+: Tal função se encontra no módulo \char`\"{}dados\char`\"{}. Num estado inicial, temos que o jogador 1 é o atual jogador; o número de jogadas é igual a zero; a peça branca se encontra na casa de coordenada e5 e todas as outras casas estão vazias. Como forma de facilitar o processo, inicializou-\/se, também, a última jogada como e5. {\bfseries{Nota\+:}} houve dificuldade em utilizar o {\itshape typedef enum} para enumerar o estado de cada casa, por isso definiu-\/se {\itshape B\+R\+A\+N\+CA}, {\itshape P\+R\+E\+TA} e {\itshape V\+A\+Z\+IA} como valores inteiros (utilizando $\ast$\#define B\+R\+A\+N\+CA 1$\ast$, por exemplo).
\item {\itshape Colocar uma peça branca na casa jogada}\+: Tal função se encontra no módulo \char`\"{}modificar\+\_\+estado\char`\"{} ({\itshape void \mbox{\hyperlink{group__sub_alpha1_ga4069ee1a3c9975d368b648b5fbff1c5c}{atualizar\+\_\+tab(\+E\+S\+T\+A\+D\+O $\ast$e, C\+O\+O\+R\+D\+E\+N\+A\+D\+A c)}}}). Consiste simplesmente em mudar o estado da casa no array e-\/$>$tab. Foi definida de forma que, assumindo que a última jogada já tenha sido atualizada, a peça preta é colocada no lugar da casa branca. Falta validar a jogada.
\item {\itshape Mostrar o tabuleiro com a jogada efetuada}\+: Tal função se encontra no módulo \char`\"{}interface\char`\"{}. A função imprime o atual estado do tabuleiro (usando o array e-\/$>$tab), por meio de um ciclo que verifica o estado de cada casa e imprime o char desejado para cada estado.
